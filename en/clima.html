Botoronga - Temperature Monitor in SP

Bot that posts daily updates with temperature variation graphs in São Paulo

@botoronga2 @botoronga.bsky.social

About Botoronga

Botoronga is an automated system that monitors and publishes daily temperature variations in São Paulo city, with informative graphs and community interactions.

Main Features:

  • Weather Data Collection:
    • Captures temperature data from reliable sources
    • Processes hourly information
    • Converts units and applies filters for better visualization
  • Graph Generation:
    • Creates clear visualizations of temperature variation
    • Applies smoothing for better readability
    • Includes informative markings and legends
  • Automated Publishing:
    • Posts daily graphs on Twitter/X and Bluesky
    • Includes current temperature at posting time
    • Platform-specific formatting
  • Community Interaction:
    • Identifies and interacts with posts about temperature in SP
    • Likes and reposts relevant content
    • Respects API rate limits

Technologies Used:

Python Selenium Matplotlib Tweepy Bluesky API OpenWeatherMap API

How It Works

Data Collection and Processing Flow:

  1. Data Collection: Accesses weather sources to obtain historical temperature data
  2. Processing:
    • Converts temperatures from Fahrenheit to Celsius
    • Groups data by hour
    • Applies smoothing filter for better visualization
  3. Graph Generation: Creates visualization with Matplotlib including:
    • Title with current date
    • Clearly marked axes
    • Grid for easy reading
  4. Current Temperature: Queries OpenWeatherMap API to get temperature at posting time
  5. Publishing: Sends the graph and information to Twitter/X and Bluesky

Automated Interaction:

  • Searches for posts with hashtags related to temperature in SP
  • Filters inappropriate content or suspicious accounts
  • Records interactions to avoid duplicates
  • Respects API rate limits with exponential backoff

Monitoring and Control

Botoronga includes advanced systems to ensure continuous operation within platform limits:

  • Interaction Logging: Stores all actions to avoid duplicates
  • Rate Limits: Respects API rate limits with precise counting
  • Exponential Backoff: Intelligent pauses when limits are reached
  • Notifications: Alert system for operational issues
