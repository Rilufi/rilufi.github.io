Sobre o Botoronga

O Botoronga é um sistema automatizado que monitora e publica diariamente a variação de temperatura na cidade de São Paulo, com gráficos informativos e interações com a comunidade.

Principais Funcionalidades:

  • Coleta de Dados Meteorológicos:
    • Captura dados de temperatura de fontes confiáveis
    • Processa informações horárias
    • Converte unidades e aplica filtros para melhor visualização
  • Geração de Gráficos:
    • Cria visualizações claras da variação de temperatura
    • Aplica suavização para melhor legibilidade
    • Inclui marcações e legendas informativas
  • Publicação Automatizada:
    • Posta gráficos diários no Twitter/X e Bluesky
    • Inclui a temperatura atual no momento da postagem
    • Formatação específica para cada plataforma
  • Interação com a Comunidade:
    • Identifica e interage com posts sobre temperatura em SP
    • Curtir e repostar conteúdo relevante
    • Respeita limites de taxa das APIs

Tecnologias Utilizadas:

Python Selenium Matplotlib Tweepy Bluesky API OpenWeatherMap API

Como Funciona

Fluxo de Coleta e Processamento:

  1. Coleta de Dados: Acessa fontes meteorológicas para obter dados históricos de temperatura
  2. Processamento:
    • Converte temperaturas de Fahrenheit para Celsius
    • Agrupa dados por hora
    • Aplica filtro de suavização para melhor visualização
  3. Geração do Gráfico: Cria visualização com Matplotlib incluindo:
    • Título com data atual
    • Eixos claramente marcados
    • Grade para fácil leitura
  4. Temperatura Atual: Consulta API do OpenWeatherMap para obter a temperatura no momento da postagem
  5. Publicação: Envia o gráfico e informações para Twitter/X e Bluesky

Interação Automatizada:

  • Busca posts com hashtags relacionadas a temperatura em SP
  • Filtra conteúdo inapropriado ou de contas suspeitas
  • Registra interações para evitar duplicações
  • Respeita limites de taxa das APIs com backoff exponencial

Monitoramento e Controle

O Botoronga inclui sistemas avançados para garantir operação contínua e dentro dos limites das plataformas:

  • Registro de Interações: Armazena todas as ações para evitar duplicações
  • Limites de Taxa: Respeita os rate limits das APIs com contagem precisa
  • Backoff Exponencial: Pausas inteligentes quando limites são atingidos
  • Notificações: Sistema de alerta para problemas operacionais
